# Contributing to GhostSurf

Thank you for considering contributing to GhostSurf! Your time and effort are highly valued. This guide will help you get started.

---

## 📌 Ways to Contribute

You can help improve GhostSurf in several ways:

- Reporting bugs
- Submitting feature requests
- Improving documentation
- Developing new features or enhancements
- Reviewing and testing pull requests

---

## 📦 Getting Started

1. **Fork this repository** by clicking the "Fork" button at the top of the page.
2. **Clone your fork** to your local machine:

       git clone https://github.com/your-username/ghostsurf.git
       cd ghostsurf

3. **Create a new branch** for your work:

       git checkout -b feature/your-feature-name

4. **Make your changes locally.**
5. **Commit your changes** with a descriptive message:

       git commit -m "feat: add useful feature"
       
6. **Push to your fork:**

       git push origin feature/your-feature-name

7. **Open a Pull Request (PR)** from your branch on your fork to the main branch of this repository.

---

## 📝 Code Style & Practices

- Follow existing code patterns and naming conventions.
- Keep your code clean, modular, and well-documented.
- Comment complex sections when necessary.
- Ensure your changes do not break existing functionality.

---

## 🐛 Reporting Issues

If you encounter a bug or have a feature suggestion:

1. Check if it’s already reported in the [issues tab](https://github.com/tamrinotte/ghostsurf/issues).
2. If not, [open a new issue](https://github.com/tamrinotte/ghostsurf/issues/new/choose).
3. Provide a clear and descriptive title and explain the problem or idea in detail.
4. Include steps to reproduce the issue if applicable.

---

## 📌 Labels for New Contributors

Look for issues labeled:
- `good first issue`
- `help wanted`

These are good starting points if you're new to the project.
